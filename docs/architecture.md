# Архитектура SisterMap

## Схема

```
data.json ──► map.py ──┬──► sister_map.html
                       └──► logs/sister_map.log
```

## Компоненты

### 1. data.json (вход)
Исходные данные: 33 записи с полем `code` (ISO alpha-2) для привязки к координатам.

### 2. map.py (обработчик)
Скрипт на Python состоит из функциональных блоков:

| Функция             | Назначение                                        |
|---------------------|---------------------------------------------------|
| `setup_logging()`   | Настройка логгирования (stdout + файл)            |
| `download_geojson()`| Скачивание границ стран (Natural Earth), если нет |
| `load_data()`       | Чтение и валидация data.json                      |
| `create_map()`      | Создание карты Folium с GeoJson-полигонами        |
| `main()`            | Точка входа: сборка шагов                         |

Логика карты:
- Читает `data.json`
- Скачивает/загружает `countries.geojson` (Natural Earth 110m)
- Обогащает GeoJSON-свойства данными из data.json (по ISO A2→A3)
- Рисует полигоны стран, залитые цветом языковой группы
- Тултип при наведении: страна → слово
- Popup по клику: слово, транскрипция, группа, справка
- Сохраняет карту в `sister_map.html`

### 3. sister_map.html (выход)
Автономный HTML-файл на Leaflet (JS). Открывается в браузере без сервера.

### 4. logs/sister_map.log (выход)
Файл лога с дописыванием (append mode). Уровни: INFO, ERROR.
Папка `logs/` создаётся автоматически при запуске, если отсутствует. Исключена из git через `.gitignore`.

## Зависимости
- Python 3.8+
- Folium (Leaflet.js обёртка)
- Нет внешних API или БД
